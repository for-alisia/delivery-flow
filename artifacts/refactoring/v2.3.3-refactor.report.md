## Implemented Refactoring

1. Enriched orchestration change model contract:
   - Added `ChangeField` enum with `LABEL`.
   - Updated `Change` interface to require `field()`, `id()`, `name()`.
   - Updated `LabelChange` to return `ChangeField.LABEL`.

2. Aligned DTO contract to domain structure:
   - `IssueDetailDto.ChangeSetDto` changed from empty marker to sealed interface with explicit accessors.
   - Added `IssueDetailDto.ChangeDto` sealed interface and made `LabelChangeDto` implement it.
   - Changed `LabelChangeDto.field` type from `String` to `ChangeField`.

3. Parallelized issue detail orchestration flow:
   - `IssuesService.getIssueDetail` now fetches issue detail and label events concurrently using `CompletableFuture`.
   - Added completion exception unwrapping to preserve typed runtime exceptions.
   - Added composition duration logging and consolidated completion log.

4. Improved orchestration search logging:
   - `IssuesService.getIssues` now logs actual filter values (`state`, `label`, `assignee`, `milestone`) instead of presence booleans.

5. Expanded mapper test coverage:
   - Added missing required-field tests for label event mapping (`user`, `action`, `created_at`).
   - Added multi-event ordering test for label events.
   - Added multi-assignee and nullable-assignee-field coverage in issue-detail mapper tests.

6. Updated runtime/component behavior checks for parallel fetch:
   - Component tests now validate that label-events endpoint is called during parallel get-single fetch even when issue detail is 404.
   - Added label-events 404 stub support helper.

7. Updated project metadata/docs:
   - Updated context map entries for `ChangeField` and DTO inner contract shape.
   - Updated issue HTTP note for enum-backed `change.field`.

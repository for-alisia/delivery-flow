## 3) Deviations From Review Recommendations

- The initial default failsafe exclude for `*KarateTest.java` was removed after validation because it was inherited by the `karate` profile execution and prevented Karate tests from running. Default `mvn verify` remains isolated from Karate via explicit failsafe include patterns (`*IT.java`, `*ComponentTest.java`).
- `scripts/karate-test.sh` intentionally runs `mvn verify -Pkarate`, which executes the full verify pipeline plus the Karate execution. This preserves existing quality behavior and adds opt-in Karate coverage.

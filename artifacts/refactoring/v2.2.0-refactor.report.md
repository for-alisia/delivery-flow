## 2) Implemented Refactoring

### Core architecture consolidation
- Added `IssuesPort` with both use-cases (`getIssues`, `createIssue`).
- Added `IssuesService` with both use-cases.
- Removed split interfaces/services:
  - `GetIssuesPort`
  - `CreateIssuePort`
  - `GetIssuesService`
  - `CreateIssueService`

### Controller dependency reduction
- Updated `IssuesController` to inject:
  - `IssuesService`
  - `IssuesRequestMapper`
  - `IssuesResponseMapper`
- Removed controller dependency on split create mappers/services.

### Mapper consolidation
- `IssuesRequestMapper` now includes create mapping (`toCreateIssueCommand`).
- `IssuesResponseMapper` now includes create response mapping (`toCreateIssueResponse`).
- Removed:
  - `CreateIssueRequestMapper`
  - `CreateIssueResponseMapper`

### Integration adapter consolidation
- Consolidated create+search behavior into `GitLabIssuesAdapter` implementing `IssuesPort`.
- Removed `GitLabCreateIssueAdapter`.
- Kept layer boundaries intact (integration types remain within integration package).

### Exception semantics
- Updated `GitLabExceptionMapper` messages from retrieve-specific wording to neutral operation wording:
  - `GitLab <resource> operation failed`
